Meshtastic Basics
MeshCore

Radio settings need to match.

MeshCore nodes using different frequency, bandwidth, spreading factor, or coding rate settings will not participate in the same radio network.

For local use, start with the USA/Canada recommended preset shown above rather than entering custom radio parameters manually.

Flood adverts

Hampden County Mesh recommends a 47 hour flood advert interval where the setting is exposed. This is a local starting recommendation for fixed MeshCore infrastructure.

Repeater advert path hashes

Repeaters running current MeshCore firmware can use longer path hashes in their own adverts. This helps mapping and analysis tools distinguish repeaters whose shorter identifiers may otherwise collide.

For a repeater on firmware 1.14 or newer, this CLI command selects a 2-byte path hash for the repeater's own adverts:

set path.hash.mode 1

This setting controls the repeater's advert path hash size. It does not determine which supported path sizes the repeater forwards.

Meshtastic

Most users should stay close to the defaults.

For normal local participation, use the United States region, LongFast, the default primary channel and key, and the CLIENT role.

Infrastructure roles

ROUTER, REPEATER, and other specialized roles change how a device participates in packet forwarding. They should be used for intentionally planned infrastructure rather than ordinary handheld, vehicle, desk, or portable nodes.

Read the Meshtastic Basics page before changing device roles.

Observed Activity

Maps are useful after the radio is configured.

The Hampden County Mesh live map shows geolocated MeshCore activity observed through participating systems.

Maps help show observed activity, but a point on a map does not necessarily mean your device can directly reach that node from its current location.

Troubleshooting

If you hear nothing, check the settings first.

  • Confirm you are using the correct region or preset.
  • Confirm all MeshCore radio parameters match.
  • For Meshtastic, confirm LongFast, the default primary channel, and AQ==.
  • Make sure the antenna is attached before transmitting.
  • Restart the device or app if the connection between them is acting strangely.
  • Try another location before assuming the radio is not working.
