Recommended Settings

Start here when configuring a MeshCore or Meshtastic node for normal use around Hampden County and nearby Western Massachusetts.

Short version

Use the common regional settings first.

Hampden County Mesh recommends MeshCore for new users who want dependable local messaging. Meshtastic users are welcome, but both systems should start simple: get one normal node working before changing roles, MQTT, power, or advanced behavior.

Meshtastic Basics
MeshCore notes

MeshCore settings must match exactly.

MeshCore nodes need matching radio parameters to hear each other. A small mismatch in frequency, bandwidth, spreading factor, coding rate, or region can make a node appear silent even when nearby activity exists.

Channels

New users can start in Public. For regional Massachusetts chat, look for #ma-mesh. For testing, use #test when available. Hampden County Mesh-specific channel conventions should be confirmed in Discord before being treated as final.

Personal nodes

Most people should begin with a normal personal node or companion-style setup. Get basic send, receive, and placement working before changing advanced behavior.

Repeaters and path hashes

Fixed relay-style MeshCore nodes should use the same local radio settings. Repeater deployments in nearby Massachusetts MeshCore examples use 2-byte path hashes.

If you are intentionally configuring a MeshCore repeater and your firmware/app exposes the CLI, the command is:

set path.hash.mode 1

Do not start with repeater behavior just because the option exists. A poorly placed repeater can add noise without improving useful coverage.

Meshtastic notes

Most Meshtastic users should stay on LongFast and CLIENT.

Meshtastic is easiest to start with when you leave the common defaults alone. In this area, normal users should generally use the United States region, the LongFast / Long Range Fast preset, the default LongFast primary channel, and the default key.

Do not start by changing roles

For most users, CLIENT is the correct role. Infrastructure roles such as ROUTER, REPEATER, ROUTER_CLIENT, or ROUTER_LATE should be reserved for unusual fixed sites with good height, power, antenna placement, and a clear reason.

Location settings

Location sharing is useful for maps and testing, but exact home locations do not need to be public. Use approximate location settings when appropriate, especially for a home node.

MQTT

Normal users do not need MQTT to participate. MQTT should be treated as map and observer infrastructure, not as a beginner setup step.

Future map infrastructure

Future Hampden County Mesh MQTT should be uplink-only observer infrastructure.

Hampden County Mesh expects any future MQTT or map infrastructure to follow an uplink-only model. The point of a future broker would be to let stable observer or gateway nodes report local traffic into site-maintained maps and activity tools.

The point would not be to bridge gaps, replace RF coverage, or require normal users to connect to a Hampden County Mesh broker. A normal user with a normal node should be able to participate without touching MQTT.

Future access model

  • Stable fixed observers or gateway nodes only.
  • Uplink-only behavior.
  • Used for maps, observed activity, and analysis.
  • Not required for ordinary MeshCore or Meshtastic users.
  • Credentials and setup instructions should be handled separately once the infrastructure exists.

Until a broker and map pipeline are ready, do not publish Hampden County Mesh MQTT connection settings here.

What not to change first

A working simple node is better than a complicated broken one.

  • Do not change frequency, bandwidth, spreading factor, coding rate, or region unless you know exactly why.
  • Do not use Meshtastic router or repeater roles for a normal handheld, desk, car, or backpack node.
  • Do not enable MQTT just because the setting exists.
  • Do not publish exact home locations when approximate location is enough.
  • Do not assume maximum transmit power is always better.
  • Do not judge coverage from one indoor test. Try a window, porch, yard, vehicle, park, hill, or other real location.
Quick troubleshooting

If you hear nothing, check the simple things first.

  • Confirm you are using the correct region.
  • Confirm all MeshCore radio parameters match exactly.
  • For Meshtastic, confirm LongFast/default and AQ==.
  • Make sure the antenna is attached before transmitting.
  • Try a better location, especially near a window or outside.
  • Charge the device and restart the app if the device connection is acting strange.
  • Ask in Discord and include your device model, firmware, rough town/area, antenna, and what settings you used.
