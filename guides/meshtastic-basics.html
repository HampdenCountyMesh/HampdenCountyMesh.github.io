What Meshtastic is

Meshtastic is open-source firmware and software for LoRa radio devices. Supported nodes can exchange text messages, position information, telemetry, and other small packets over radio.

A typical personal node connects to a phone or computer through a Meshtastic client app.

How Meshtastic fits here

Hampden County Mesh supports both MeshCore and Meshtastic. MeshCore is the main network and infrastructure focus of the local project.

Meshtastic remains useful for personal nodes, portable use, telemetry, experimenting with LoRa hardware, and people who already participate in Meshtastic networks.

The two systems use different firmware, apps, settings, and network behavior. A Meshtastic node does not participate directly in a MeshCore mesh.

Device roles

For most personal Meshtastic nodes, CLIENT is the normal starting role.

Meshtastic also includes specialized roles for base stations, trackers, sensors, and network infrastructure. These roles change device behavior and should be selected for a specific use rather than because they sound more powerful.

Role definitions can change as Meshtastic develops. Use the current official device-role documentation before configuring infrastructure.

Location sharing

Meshtastic can share position information over configured channels. Think about how precise that information needs to be before broadcasting a home or other private location.

Meshtastic supports position precision controls that can reduce the accuracy of shared location data.

MQTT

MQTT lets a Meshtastic node or client forward selected traffic through an MQTT broker when an internet connection is available.

It can support integrations, remote meshes, and map reporting, but it is not required for normal Meshtastic radio use.

Leave MQTT off unless you are intentionally connecting the node to an MQTT service and understand which channels or data are being forwarded.

Quick troubleshooting

If a Meshtastic node seems quiet, check the basics first.

  • Confirm the United States region is selected.
  • Confirm the modem preset matches the nodes you expect to hear.
  • Make sure the antenna is attached before transmitting.
  • Check that the phone or computer is still connected to the node.
  • Leave the node powered long enough to hear normal activity.
  • Try another location before assuming the radio is not working.
