Meshtastic Basics

Meshtastic is a common way people first get into LoRa mesh radio. Hampden County Mesh recommends MeshCore for people starting fresh with the local direction here, but Meshtastic is still useful, welcome, and worth understanding.

What It Is How It Fits Start Simple Settings MQTT Sharing

What Meshtastic is

Meshtastic is open-source firmware and software for small LoRa radio devices. It lets supported devices send text messages and basic data over radio without relying on cell service or a normal internet connection.

A lot of people find mesh radio through Meshtastic first. The apps are familiar, the hardware options are broad, and there are active users in many places.

How it fits with Hampden County Mesh

Hampden County Mesh supports both MeshCore and Meshtastic. MeshCore is the main setup focus here because it fits the local direction for two-way messaging, fixed nodes, repeaters, and shared community use.

Meshtastic still has a place. It is good for learning how LoRa behaves, testing antennas and placement, comparing devices, telemetry, portable use, outdoor experiments, and casual participation.

You do not need to pick a side before joining the Discord. If you already have Meshtastic hardware, ask about it. If you are starting from scratch and want to follow the local Hampden County Mesh direction, start with MeshCore.

Start simple

Start with one normal Meshtastic node. Get it working, learn the app, and see what your location can actually hear before building anything more complicated.

  1. Use supported hardware.
  2. Follow the official Meshtastic getting-started docs.
  3. Use the local recommended Meshtastic starting settings.
  4. Use a clear node name that does not expose private information.
  5. Place the device somewhere reasonable, such as near a window.
  6. Ask in Discord if you are unsure what you are seeing.

For antennas, placement, mobile use, fixed nodes, and practical setup choices, read Nodes, Repeaters, and Observers.

Use simple starting settings

For local use, keep Meshtastic simple at first: United States region, normal LongFast/default channel behavior, normal public defaults, and the CLIENT role for most users.

Do not begin with router roles, repeater roles, custom channel plans, or MQTT unless you know why you are using them.

Meshtastic and MeshCore are different

Meshtastic and MeshCore both use LoRa radios, but they are not the same system. They use different apps, firmware, settings, roles, and network behavior.

Some radio lessons carry over. Height matters, placement matters, terrain matters, antennas matter, and a map point does not automatically prove direct radio range from where you are.

Maps and MQTT

Meshtastic maps and MQTT can be useful, but they are easy to misread. A device appearing on a map does not always mean it is reachable by direct local radio from your location.

Internet-connected systems, MQTT, privacy settings, delayed data, and device roles can all affect what you see. For normal beginner use, leave MQTT off and focus on local radio first.

Before buying hardware

A device that fits how you will actually use it is better than a device that looks good on paper but never gets used.

Home use

Learning at a desk or window

A simple supported device may be enough if you mainly want to learn the app and see what is nearby.

Portable use

Carrying it around

Think about battery life, case, antenna, screen, Bluetooth/app support, and charging.

Fixed use

Leaving it running

Think about power, heat, safe placement, antenna location, and whether it can stay on reliably.

Advanced use

Outdoor, router, MQTT, or map use

Ask first. Site quality, privacy, and configuration matter more than just owning the hardware.

Before posting screenshots or setup details

Meshtastic can expose more than people expect through node names, map settings, screenshots, telemetry, position sharing, and MQTT. Before posting publicly, check for private messages, exact locations, channel details, keys, and anything someone else did not agree to share.
