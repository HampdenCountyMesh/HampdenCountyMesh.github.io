MeshCore Basics

MeshCore is the main mesh focus for Hampden County Mesh. This page explains what it is, why HCM leans toward it, and where to go next for settings, flashing, node setup, and local help.

What MeshCore is

MeshCore is an off-grid messaging system for LoRa radio devices. In plain terms, it lets small radios send short messages without relying on cell service, Wi-Fi, or a normal internet connection.

A basic MeshCore setup can be simple: one supported device, the right firmware, a phone or computer app, a working antenna, and compatible local radio settings.

Why HCM focuses on MeshCore

Hampden County Mesh leans toward MeshCore because it fits the direction we are building around: local channels, fixed sites, repeaters, observers, maps, and regional mesh activity across Western Massachusetts and nearby areas.

This does not mean Meshtastic is ignored. Meshtastic is still useful and welcome. A lot of the practical radio lessons carry over: antenna placement, node height, terrain, batteries, firmware, maps, and what devices can hear from different places.

If you are starting from scratch and want to follow the local HCM direction, start with MeshCore. If you already have Meshtastic hardware, use it and ask how it fits in.

How to start

Do not start with a complicated build. Start with one normal device and learn what it can hear.

  1. Pick a supported MeshCore device.
  2. Flash the correct firmware for your device and intended role.
  3. Use the HCM recommended MeshCore settings.
  4. Pair the device with the MeshCore app or client you are using.
  5. Place the node somewhere reasonable, such as near a window.
  6. Ask in Discord if you are unsure what you are seeing.

For device roles, antennas, power, placement, repeaters, observers, maps, and wardriving, use Node Setups.

Use the local settings

MeshCore settings need to match the mesh you are trying to reach. If the frequency, bandwidth, spreading factor, or other radio settings do not match, your device may be working but listening in the wrong place.

Local terrain matters

Hampden County is not flat or simple. Hills, valleys, wooded areas, dense neighborhoods, old buildings, river corridors, farms, and high spots can all change what a small LoRa radio can hear.

Small changes can matter. A node near a window may hear more than the same node in the middle of a room. A modest antenna in a better spot may work better than a more powerful setup in a poor location.

Maps and observed activity

Observers, analyzers, dashboards, logs, and public feeds are useful, but they can be misunderstood. A map point or observed activity does not always prove that your portable node can directly reach that device from your current location.

Treat maps as clues, not guarantees. For normal local use, the useful question is still simple: what can your device hear from where it is right now?

Before buying hardware

A simple first node that actually gets used is better than a complicated setup that never leaves the planning stage.

  • If you want to learn the app and send messages, start with a companion or personal node.
  • If you want something portable, think about battery life, case, antenna, screen, and how you will carry it.
  • If you want a fixed node, think about power, safe mounting, heat, antenna placement, and whether you can leave it running.
  • If you want a repeater, observer, or outdoor node, ask first. Site quality matters more than just owning the hardware.

Privacy basics

MeshCore can create useful screenshots, map results, logs, and public activity. Check those before posting them publicly.

  • Do not post private keys, passwords, broker credentials, or Wi-Fi details.
  • Do not post exact private node coordinates without permission.
  • Do not share someone else’s setup details without permission.
  • Check screenshots for private messages, names, keys, coordinates, or admin URLs.
  • Use general areas for public discussion unless exact location sharing is clearly safe.
