What MeshCore is

MeshCore is a lightweight messaging and packet-routing system for LoRa radio devices. Compatible nodes can communicate directly or through other MeshCore infrastructure without relying on normal internet or cellular service.

A basic setup can be simple: one supported device, MeshCore firmware, a phone or computer app, a working antenna, and radio settings that match the regional mesh.

Start with a companion

For most people, the first MeshCore device should be a companion or personal node connected to a phone or computer.

Get one device working before planning repeaters, room servers, observers, solar installations, or other infrastructure.

Companions

A companion is the normal place to begin. It lets you discover contacts, use channels, send messages, and learn what the local mesh looks like.

Repeaters

Repeaters help relay MeshCore traffic. Their usefulness depends heavily on placement, antenna setup, power, and the radio paths around them.

Running a repeater is infrastructure work rather than a normal beginner step.

Building Better Mesh Coverage

Room servers

A room server provides a persistent room for group messages. It serves a different purpose from a repeater, and the two are normally better treated as separate infrastructure roles.

Observers

An observer is supporting infrastructure around the mesh rather than a replacement for a repeater. It reports what it hears to other tools but does not improve the RF path between two radios.

Hampden County Mesh uses observed packet activity to support its live map and activity analyzer.

MeshCore channels

Channels are used for group messages. Public and shared channels use flood routing, so a channel name should not be treated as a way to keep traffic physically confined to one town or county.

Public

The broad public channel. Useful for first contact and occasional messages intended for the wider mesh.

Hashtag channels

Public topic or regional channels such as #test. They help organize conversation but are still public channel traffic.

Shared channels

Channels configured with a shared secret. Anyone with that channel configuration can participate and read its messages.

Local channel use

Use #test for routine test messages where it is configured rather than repeatedly testing in Public.

Hampden County Mesh also uses or is preparing channels for local tools and community purposes. Current channel information is kept in the Discord so conventions can change without leaving old instructions scattered across the site.

Keep public messages public-safe

Public or shared channels should not be treated as private messaging. Avoid posting information that does not belong in a broadly visible group conversation.

Sharing Safely

Maps and observed activity

The Hampden County Mesh live map shows geolocated MeshCore activity heard through participating observer systems.

The activity analyzer provides a deeper view of packets, nodes, paths, observers, and other data collected by the HCM observer feed.

Observed activity is not a complete list of every MeshCore device in the region. A node must be heard by a participating observer, and location data must be available for it to appear on the live map.

A visible node also does not necessarily mean your own device can reach it directly from its current location.

MQTT and observer infrastructure

Normal MeshCore users do not need MQTT to participate.

Hampden County Mesh uses MQTT behind its observer, map, and analyzer systems. That is infrastructure for collecting and presenting observed activity rather than a requirement for a personal node.

Quick troubleshooting

If a MeshCore device seems quiet, check the simple things first.

  • Confirm the regional radio settings match.
  • Make sure the correct firmware was installed for the device.
  • Make sure the antenna is attached before transmitting.
  • Leave the device powered long enough to hear normal activity.
  • Try another location before assuming the radio is not working.
  • Change one thing at a time while troubleshooting.
