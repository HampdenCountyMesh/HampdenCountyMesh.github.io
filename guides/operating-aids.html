Operating Aids

These are simple checklists and templates for local mesh testing. They are meant to make signal checks, field notes, device notes, wardriving notes, and Discord reports easier to compare later.

Keep public notes useful but careful. Do not post private keys, passwords, exact home locations, private coordinates, Wi-Fi details, broker credentials, admin URLs, or anything someone did not agree to share.
Quick Check Field Notes Discord Report Node Note Wardriving Before Posting

How to use this page

You do not need to fill out every line every time. Use the parts that make the result easier for someone else to understand.

The main idea is simple: write down what you tested, where you tested in general terms, what equipment you used, what changed, and what happened.

These templates are especially useful when comparing notes in Discord, GitHub issues, local documentation, or future public pages.

Quick signal check

Use this when you just want to see whether another person or device can hear you.

  1. Charge the device or connect stable power.
  2. Put the device in a reasonable spot, such as near a window or outside if safe.
  3. Confirm the device name is recognizable and not too personal.
  4. Confirm you are using the expected MeshCore or Meshtastic setup.
  5. Ask someone nearby to listen for a test message.
  6. Send a short test message.
  7. Write down whether it was heard, not heard, weak, delayed, or unclear.
  8. Change one thing and test again.

Simple message tests

Test messages should be short and clear. Include enough information to identify the test without posting private details.

Basic test

“Test from Westfield area portable node. Please reply if heard.”

Location comparison

“Window test from general Chicopee area. Moving outside next.”

Signal check

“Signal check near Mount Tom area. Looking for nearby confirms.”

End of test

“Test complete. Thanks for replies.”

Good operating habits

  • Use clear, short messages during tests.
  • Say when a test starts and when it ends.
  • Write down the general area and setup.
  • Change one thing at a time.
  • Do not assume a failed test means nothing works.
  • Do not assume a map point means direct radio coverage.
  • Keep private details out of public posts.
  • Ask before publishing someone else’s setup details.
  • Do not treat mesh radio as a replacement for 911.
  • Do not handle devices while driving.

Before posting public notes

Before posting a test result, photo, map, screenshot, or setup note, check it for private information.

  • Does it show an exact private home location?
  • Does it include private coordinates?
  • Does it show a password, key, broker credential, or Wi-Fi detail?
  • Does it reveal a private IP address, admin URL, or backend detail?
  • Does a photo reveal a private address, plate, mail, screen, or reflection?
  • Did everyone involved agree to have the details shared?
  • Would a town, park, or general area be enough instead?

Useful public notes do not need exact private details. Broad descriptions are often enough.
