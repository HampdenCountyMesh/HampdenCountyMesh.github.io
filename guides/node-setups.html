Node Setups

This is the practical node guide for Hampden County Mesh. It covers common MeshCore and Meshtastic setups: personal use, fixed locations, mobile use, repeaters, observers, placement, antennas, maps, MeshMapper for MeshCore coverage checks, and basic troubleshooting.

Start with one normal node

A beginner setup should be boring. One device, correct firmware, a working antenna, reasonable power, and the recommended settings are enough to start learning.

Do not begin with a repeater, router role, MQTT, outdoor solar box, custom channel plan, or multiple devices unless you already know what problem you are solving.

Use Recommended Settings before changing advanced behavior.

Common setup types

The exact names vary between MeshCore and Meshtastic, but these are the practical setup types that matter locally.

Beginner

Personal node

A normal device you use at home, carry with you, place near a window, bring in a vehicle, or use while testing. This is where most people should start.

  • Good for learning.
  • Good for first messages and tests.
  • Does not need special infrastructure settings.
Stable location

Fixed node

A node that usually stays in one place, such as a home, office, garage, workshop, club space, library, or other stable location.

  • Good for consistent local presence.
  • Benefits from good placement and stable power.
  • Should avoid exposing exact private locations publicly.
Testing

Mobile or field node

A node used from a vehicle, bag, park, hill, sidewalk, event table, or temporary test location.

  • Good for coverage checks.
  • Helps show what works from real places.
  • Can be useful before planning fixed installs.
Infrastructure

Repeater or relay-style node

A fixed node intended to help other nodes reach farther. This only helps when the location, antenna, power, and settings are suitable.

  • Needs a useful site.
  • Needs stable power.
  • Should not be used casually from poor locations.
Map support

Observer or gateway

A node or support system used to hear activity and contribute to maps or analysis. Future Hampden County Mesh MQTT access should be uplink-only for this kind of use.

  • Helps show observed activity.
  • Does not automatically improve RF coverage.
  • Should be stable and well placed before contributing data.
Backend

Map, broker, or support system

A computer, script, broker, logger, dashboard, or website tool that supports mesh visibility. These are tools around the mesh, not ordinary user node roles.

  • Useful for infrastructure work.
  • Not required for normal users.
  • Should stay separate from beginner setup.

MeshCore setup notes

MeshCore setups can behave differently depending on the role. A companion node, repeater, observer, and room/server-style support setup do not all do the same job.

  • Use the local MeshCore settings from Recommended Settings.
  • Start with a normal personal or companion-style setup before thinking about repeaters.
  • Fixed repeaters should use suitable sites, reliable power, and documented settings.
  • Observers and gateways should be treated as map/activity infrastructure, not normal beginner setup.
  • For MeshCore coverage checks, use MeshMapper.

Meshtastic setup notes

Meshtastic is easiest to start with when normal users stay on common defaults. For local use, most Meshtastic users should use LongFast/default and the CLIENT role.

  • Use the local Meshtastic settings from Recommended Settings.
  • Use CLIENT for normal handheld, desk, vehicle, or backpack use.
  • Do not use router or repeater roles unless the site is unusually useful and you understand the tradeoffs.
  • Think carefully before publishing exact home locations.

Repeaters and relay-style nodes

A repeater is not automatically helpful. A poorly placed repeater can add noise, confusion, or traffic without improving useful coverage.

A good repeater or relay-style node usually has:

  • A useful location with better-than-average height or line of sight.
  • Stable power.
  • A suitable antenna and feedline setup.
  • Weather protection if installed outdoors.
  • Permission to be installed where it is installed.
  • Documented firmware and settings.
  • Someone responsible for checking it later.

For MeshCore repeaters, use the regional radio settings and follow the path-hash note in Recommended Settings.

Observers, gateways, and future map contributors

An observer or gateway helps show what can be heard from a location. It may feed maps, logs, dashboards, or other activity tools. It does not automatically make the RF mesh stronger.

Hampden County Mesh plans to follow an uplink-only model for future MQTT and map infrastructure. That means stable observers and gateway nodes may eventually send observed activity to site-maintained systems, but normal users should not need MQTT to participate.

  • Good observers should be stable and well placed.
  • Observer data should be described as observed activity, not ownership of the whole network.
  • Credentials and broker details should not be posted publicly.
  • Normal users should focus on local settings and real RF testing first.

Antenna, power, and placement

The node itself is only part of the setup. Antenna, power, and placement often decide whether a setup is useful.

Antenna

  • Use an antenna intended for the correct band.
  • Attach the antenna before transmitting.
  • Keep tiny stock antennas in perspective. They are convenient, not magic.
  • A better antenna helps only when it is matched, mounted, and placed well.

Power

  • Battery nodes are useful for testing and portable use.
  • Fixed nodes should have stable power and reasonable cable management.
  • Outdoor or solar nodes need more planning than a desk node.
  • Power problems can look like radio problems.

Placement

  • Height usually helps.
  • A window can be better than the middle of a room.
  • Outside can be better than inside, if the setup is safe.
  • Dense buildings, hills, valleys, trees, and terrain all matter.
  • Move away from large metal objects, appliances, and dense walls when practical.
  • Change one thing at a time so the result means something.

Field testing

Field testing means trying a node from real places and seeing what changes. A useful test can be as simple as trying a window, porch, park, hill, parking lot, vehicle, or different part of town.

Simple field test

  1. Use the recommended settings.
  2. Use a known device and antenna setup.
  3. Try a real location with better placement or clearer surroundings.
  4. Leave the node running long enough to observe activity.
  5. Try one change, such as higher placement or a different antenna.
  6. Compare what changed.

A quiet result can still be useful. It may mean the location, antenna, settings, timing, or nearby activity were not favorable.

MeshMapper for MeshCore coverage checks

For MeshCore field testing and coverage mapping, use MeshMapper. It is the main tool we point people toward for collecting and viewing MeshCore reception data.

Keep it safe. Do not handle devices while driving. Have a passenger manage equipment when possible, keep devices mounted or stored safely, and avoid posting private addresses or exact private locations.

Maps and observed activity

Maps, analyzers, observers, MQTT feeds, logs, and dashboards can be useful, but they can also be misunderstood. A map point or recently observed device does not always mean you can reach that device directly over radio from where you are.

Some information may come from an observer. Some may come through an internet-connected path. Some may be delayed, incomplete, temporary, mobile, or outside Hampden County.

Use maps as clues, not guarantees. For normal local use, the useful question is still simple: what can your device hear from where it is right now, and can someone nearby hear you back?

Sharing useful results

Sharing what worked can help other people, but it does not need to become a formal report. A short Discord post is usually enough.

Useful public examples

  • “MeshMapper showed MeshCore activity along part of Route 20.”
  • “Window placement worked better than desk placement in Westfield.”
  • “A higher spot worked better than lower ground.”
  • “Outdoor antenna placement helped more than changing power.”
  • “A mobile test was quiet in one area but active closer to higher ground.”

Less useful examples

  • “It does not work.”
  • “Coverage is bad everywhere.”
  • “This device is the best.”
  • “I saw a map point so I know I can reach it.”

Specific, limited observations are more useful than broad claims. Keep exact private locations out of public posts unless there is a clear reason and permission to share them.

Location and privacy

Useful mesh information does not require exposing private addresses. For most public posts, use a general location description.

Usually okay for public posts

  • Town or general area.
  • Public landmark area.
  • General terrain notes, such as hill, valley, wooded area, or dense buildings.
  • General placement, such as window, vehicle roof, tree-mounted, or portable test.
  • General result, such as worked, did not work, weak, quiet, active, or reliable.

Avoid posting publicly

  • Exact home addresses.
  • Exact private coordinates.
  • Someone else’s workplace or residence.
  • Private keys, passwords, or broker credentials.
  • Wi-Fi names, Wi-Fi passwords, private IP addresses, or admin URLs.
  • Photos that reveal private details unintentionally.
  • Access instructions for private property or equipment.

Read Sharing Safely before posting public setup photos, logs, screenshots, or map observations.

Photos

Photos can help with antennas, mounting, field checks, and hardware setups. They also need a little care.

  • Check that the photo does not reveal a private address.
  • Remove location metadata when needed.
  • Avoid showing keys, passwords, private screens, or personal details.
  • Check reflections, license plates, mail, signs, and screen contents.
  • Do not include people unless they agreed to be included.
  • Use a general caption that explains why the photo matters.

Troubleshooting

If nothing seems to work, check the simple things before assuming the node is broken.

  • Confirm the antenna is attached.
  • Confirm the battery or power source is stable.
  • Confirm the recommended settings match the system you are using.
  • Try a window or outdoor location.
  • Leave the node running long enough to hear normal activity.
  • Restart the app or device if the connection is acting strange.
  • Ask in Discord with device model, system, settings, antenna, and general area.
