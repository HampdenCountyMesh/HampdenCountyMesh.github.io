Radio Aids

These are simple checklists and templates for practical radio use. They are meant to help with MeshCore, Meshtastic, device setup, local radio notes, mobile use, voice radio, and safer sharing.

Keep public information useful but careful. Do not post private keys, passwords, exact home locations, private coordinates, Wi-Fi details, broker credentials, admin URLs, or anything someone did not agree to share.
Quick Node Check Device Help Local Use Mobile Use Voice Radio Before Posting

How to use this page

You do not need to fill out every line every time. Use the parts that make the situation easier for someone else to understand.

The main idea is simple: write down what you were trying to do, what equipment you used, where you were in general terms, what changed, and what happened.

These templates are especially useful in Discord, GitHub issues, local documentation, or personal notes.

Quick node check

Use this when you want a simple first look at whether a MeshCore or Meshtastic node appears to be working as expected.

  1. Charge the device or connect stable power.
  2. Put the device in a reasonable spot, such as near a window or outside if safe.
  3. Confirm the device name is recognizable and not too personal.
  4. Confirm you are using the expected MeshCore or Meshtastic setup.
  5. Leave the device powered long enough to see normal activity.
  6. Send a short message if that makes sense for your setup.
  7. Ask in Discord if you are unsure what you are seeing.
  8. Change one thing and try again.

Simple message text

Test messages should be short and clear. Include enough information to identify the test without posting private details.

Basic test

“Test from Westfield area portable node. Please reply if heard.”

Location comparison

“Window test from general Chicopee area. Moving outside next.”

Area check

“Trying a node near Mount Tom area. Looking for nearby replies.”

End of test

“Test complete. Thanks for replies.”

Good radio habits

  • Use clear, short messages during tests.
  • Say when a test starts and when it ends.
  • Write down the general area and setup.
  • Change one thing at a time.
  • Do not assume a quiet result means the device is broken.
  • Do not assume a map point means direct radio range.
  • Keep private details out of public posts.
  • Ask before publishing someone else’s setup details.
  • Do not treat mesh radio as a replacement for 911.
  • Do not handle devices while driving.

Before posting public information

Before posting a result, photo, map, screenshot, or setup note, check it for private information.

  • Does it show an exact private home location?
  • Does it include private coordinates?
  • Does it show a password, key, broker credential, or Wi-Fi detail?
  • Does it reveal a private IP address, admin URL, or backend detail?
  • Does a photo reveal a private address, plate, mail, screen, or reflection?
  • Did everyone involved agree to have the details shared?
  • Would a town, park, or general area be enough instead?

Useful public information does not need exact private details. Broad descriptions are often enough.
