Getting Started

Start here if you are new to MeshCore, Meshtastic, or local mesh networking. The goal is simple: pick a path, get one normal node working, use the recommended settings, and test from real places.

Quick Path What Mesh Is Choose a Path Settings First Test Ask for Help

Quick path

You do not need to understand every role, setting, or map tool before starting. Most people should begin with one normal personal node and keep the setup simple.

  1. Read the short explanation on this page.
  2. Choose MeshCore or Meshtastic as your first path.
  3. Use the Recommended Settings.
  4. Place the node near a window or take it somewhere with a clearer view.
  5. Use Node Setups for placement, antennas, maps, wardriving, and troubleshooting.
  6. Check Coverage and ask in Discord if you are unsure what you are seeing.

The first goal is not a perfect installation. The first goal is to get oriented, get a device working, and learn what mesh looks like from real locations around Hampden County.

What mesh is

Mesh radios pass short messages between nearby devices. A node can send and receive messages, and other nodes may help relay traffic across an area.

Coverage is local and practical. It depends on terrain, height, antenna placement, buildings, trees, and whether there are other useful nodes nearby.

Maps can help, but they do not replace real testing. A map point, dashboard result, or observed activity does not always mean your node can directly reach that device from where you are.

What you need

A first setup does not need to be complicated. For most people, the basic pieces are:

  • A supported MeshCore or Meshtastic device.
  • A phone or computer to configure it.
  • A working antenna for the device and frequency band.
  • A power source or battery.
  • A place to test it, such as a window, yard, vehicle, public park, or other real location.

Do not start with outdoor installs, solar power, MQTT, router roles, repeater behavior, or multiple nodes unless you already know why you need them.

Basic roles

For a beginner, the important distinction is simple: most people start with a normal personal node. Other roles can come later.

Personal node

A normal device you use at home, carry with you, place near a window, bring in a vehicle, or use while testing.

Fixed node

A node that usually stays in one place, such as a home, office, garage, workshop, or other stable location.

Repeater or relay-style node

A node intended to help other nodes reach farther. This only helps when the site, antenna, power, and settings are suitable.

Observer or gateway

A node or support system used to hear activity and contribute to maps or analysis. Future Hampden County Mesh MQTT access should be uplink-only for this kind of use.

Read Node Setups for practical details about node types, antennas, placement, mobile use, repeaters, observers, maps, and wardriving.

Use the recommended settings

Settings are where many beginner problems start. Use the local starting settings first. Change advanced settings only after you know what problem you are solving.

MeshCore short version

  • US / USA-Canada recommended preset
  • 910.525 MHz
  • 62.5 kHz bandwidth
  • SF7
  • CR5

Meshtastic short version

  • United States region
  • LongFast / default primary channel
  • AQ== primary key
  • CLIENT role for most users
  • MQTT off for normal beginner use

The canonical settings page is Recommended Settings. Use that page when configuring a device.

First setup goals

A good first setup is boring. Get the device running, make sure the settings are right, place it somewhere reasonable, and watch what happens.

  • Flash the correct firmware for your device.
  • Set a clear node name that does not expose private details.
  • Use the recommended settings before changing anything advanced.
  • Attach the antenna before transmitting.
  • Place the device near a window or outside temporarily if safe.
  • Make one change at a time when troubleshooting.
  • Use maps, app screens, or tools like MeshMapper where appropriate instead of trying to manually track everything.

First test

After the device is running, test simply. Do not judge the whole mesh from one indoor attempt.

  1. Put the node near a window or another reasonable location.
  2. Leave it powered long enough to hear normal activity.
  3. Send a short test message only if it makes sense for the system and channel you are using.
  4. Move the node once and see whether anything changes.
  5. Try a different real location, such as a yard, car, park, hill, or other open area.
  6. Keep location details general when posting publicly.

For placement, range, mobile use, maps, and wardriving, read Node Setups.

What affects range

More power is usually not the first answer. Placement and antenna setup often matter more.

  • Height usually helps.
  • A window can be better than the middle of a room.
  • Outside can be better than inside, if the setup is safe.
  • Terrain can block or shape radio paths.
  • A better antenna can help, but only if it is matched and installed well.
  • A stable power source matters for fixed nodes.
  • Other useful nodes nearby make testing easier.

Hampden County has hills, valleys, dense neighborhoods, wooded areas, river corridors, and high spots. Results from one town, street, building, or park may not apply everywhere else.

What to avoid at first

  • Buying several devices before understanding your first one.
  • Changing many settings at the same time.
  • Using router or repeater roles on a normal beginner node.
  • Enabling MQTT just because the option exists.
  • Publishing exact private locations when a general area is enough.
  • Assuming no replies means the device is broken.
  • Assuming a map point means you can directly reach that node.

Privacy basics

Mesh observations are more useful when they avoid exposing private details.

  • Do not post private keys, passwords, broker credentials, or Wi-Fi details.
  • Do not publish exact home node locations without careful thought and permission.
  • Do not share someone else’s setup details without permission.
  • Check photos for addresses, license plates, screens, reflections, and EXIF metadata.
  • Use a general area when talking publicly about where a device was used.

Read Sharing Safely before posting public information or photos.
