Interactive Learning

How MeshCore Works

Follow a message across a sample MeshCore network. Watch an unfamiliar direct-message destination get found by flooding, compare it with a later message using a learned path, and see what changes when a path breaks or a channel message is sent.

Start with the first message

  1. Use the sample map or generate a new one.
  2. Leave Message on First direct message.
  3. Select Run message, then follow the bold next-step prompt.

The log keeps each run on the current map so you can compare them. Generating a new map clears the log.

What It Shows

An educational model of message movement

The simulator models direct reception, repeater forwarding, flood hop progression, a learned direct-message path, channel flooding, and simplified recovery when a stored path no longer works.

Mountains block simulated radio paths. Wooded areas reduce simulated range instead of acting like solid walls. Those terrain effects make routing choices easier to see; they are not calculated from a real location.

What it does not predict

This is not a firmware emulator or an RF coverage forecast. Real results also depend on radios, antennas, elevation, buildings, interference, packet timing, firmware versions, regional settings, and which nodes are available at that moment.

Some timing and retry details are simplified so the important difference between flooding and a directed path remains readable.
