# Hampden County Mesh Discord

The Hampden County Mesh Discord is the main discussion and coordination space for MeshCore, Meshtastic, local radio, node help, field testing, infrastructure, and project work.

## Joining

Public join route:

https://hampdencountymesh.org/join/

Use the website join route instead of publishing the underlying Discord invite. This preserves source tracking and allows the invite to be changed without updating every public link.

Discord-related questions may be sent to:

[discord@hampdencountymesh.org](mailto:discord@hampdencountymesh.org)

## Current Channel Structure

Channel names may change as the community develops, but the current structure is:

### Start Here

- `welcome`
- `rules`
- `hampden-county-mesh-announcements`
- `elsewhere-announcements`
- `server-guide`
- `new-member-lobby`

These channels cover onboarding, rules, official announcements, regional announcements, and first questions.

### Community

- `general`
- `introductions`
- `questions-help`
- `radio`
- `show-your-stuff`
- `off-topic`

Use these channels for normal conversation, introductions, general help, radio topics, project photos, and casual discussion.

### Main

- `meshcore`
- `meshtastic`
- `node-help`
- `wardriving`
- `infrastructure`

Use:

- `meshcore` for MeshCore apps, firmware, node roles, channels, repeaters, observers, and regional activity
- `meshtastic` for Meshtastic devices, firmware, channels, roles, and related questions
- `node-help` for hardware, antennas, power, enclosures, placement, and troubleshooting
- `wardriving` for MeshCore field testing and MeshMapper discussion
- `infrastructure` for observers, fixed nodes, MQTT, CoreScope, hosting, and related systems

MeshMapper is used for MeshCore field testing and should not be presented as a Meshtastic tool.

Western Massachusetts MeshMapper region:

https://psf.meshmapper.net/

### Project Work

- `website`
- `docs-writing`
- `map-automation`
- `invite-bot`
- `announcements-planning`

These channels are used for work on the website, documentation, public map, invite tracking, bot development, and project announcements.

Some project channels may be limited by role when they contain unfinished work or operational details.

### Voice

`Workshop/Help Desk` is a voice channel for live help, workshops, and informal project discussion.

It is separate from text channels such as `questions-help` and `node-help`.

## First Steps

After joining:

1. Read `rules` and `server-guide`.
2. Complete onboarding when prompted.
3. Choose relevant interest and region roles.
4. Use `new-member-lobby` or `questions-help` when unsure where to begin.
5. Use `node-help`, `meshcore`, or `meshtastic` for device-specific questions.

Beginner questions are welcome. Members do not need prior radio experience or expensive equipment to participate.

## Getting Technical Help

Useful information may include:

- Device model
- Firmware version
- Mesh system
- Current role or mode
- Relevant settings
- Phone app or computer software being used
- Exact error message
- What has already been tried
- General location when terrain or regional settings matter

Remove credentials, private messages, and unnecessary location details from screenshots or logs.

## Maps and Observed Activity

The public Hampden County Mesh map is active at:

https://map.hampdencountymesh.org/

Detailed packet, path, node, and observer information is available at:

https://analyzer.hampdencountymesh.org/

The map and analyzer show observed MeshCore activity heard through participating systems. They are partial activity views and do not represent every regional node or radio path.

Map and analyzer development may be discussed in `map-automation` and `infrastructure`.

## Project Integrations

Discord integrations may be used for:

- Repository updates
- Project announcements
- Infrastructure notices
- Map or service status
- Invite tracking
- Bot testing

Automated messages should have a specific purpose and should remain separate from normal discussion where practical.

Do not create channels for speculative integrations before there is an actual workflow to support them.

## Privacy and Security

Do not post credentials, private keys, access tokens, broker passwords, private messages, or information that would provide unauthorized access.

Do not publish someone else’s exact private location or installation details without permission.

Sensitive security matters should be reported to:

[security@hampdencountymesh.org](mailto:security@hampdencountymesh.org)

Conduct and moderation expectations are defined in:

- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `docs/ASSETS.md`
- `docs/data-access-and-observed-activity.md`

## Moderation

Moderators may remove content, redirect discussions, restrict access, or remove members when necessary to enforce the server rules and Code of Conduct.

Conduct reports may be sent to:

[discord@hampdencountymesh.org](mailto:discord@hampdencountymesh.org)

Private reports should not be posted in public GitHub issues or public Discord channels.
